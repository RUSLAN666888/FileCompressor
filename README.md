### Исходные требования к проекту
Система сжатия файлов на основе Huffman + LZ77 (сложно)
Суть проекта: Реализация двух популярных алгоритмов сжатия и их сравнение на различных типах данных.
Что делать: Написать кодер и декодер для алгоритма Хаффмана (статический и адаптивный) и для алгоритма LZ77 (или LZW).
Провести тестирование на текстах, картинках (как на бинарных данных), измерить степень сжатия и скорость работы.
Сложность — в эффективной работе с битовыми потоками.

## О проекте

Проект реализует возможность создания и распаковки архивов, используя следующие алгоритмы сжатия:
- статический алгоритм Хаффмана
- адаптивный алгоритм Хаффмана
- LZ77
- LZ78

## Техническая реализация

- **Формат архивов:** Собственный формат с расширением **`.arch`**
- **Язык программирования:** **C++** (стандарт C++17)
- **Пользовательский интерфейс:** Нативная реализация на **Win32API**
- **Среда разработки:** Visual Studio 2022

**Файлы, на которых проводилось тестирование:** доступны в [Issue #1: test_files](https://github.com/RUSLAN666888/FileCompressor/issues/1#issue-3762143846).

## Сборка через Visual Studio
1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/RUSLAN666888/FileCompressor.git
    cd FileCompressor
    ```
2.  Откройте файл решения **`FileCompressor.sln`** в Visual Studio.
3.  Выберите конфигурацию сборки (например, `x64 Release`).
4.  Соберите решение: **`Ctrl + Shift + B`** или меню *Сборка → Собрать решение*.
